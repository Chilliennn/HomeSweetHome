UC-102: DECIDE YOUTH APPLICATION

UC102_x
Description
UC102_1
The system shall display interest notification to elderly when youth expresses interest
UC102_2
The system shall allow elderly to accept or decline youth interest
UC102_3
The system shall prevent elderly from accepting interest when active pre-match limit is reached
UC102_4
The system shall prevent interest acceptance when youth has reached active pre-match limit
UC102_5
The system shall create pre-match communication session when elderly accepts interest
UC102_6
The system shall notify youth when elderly declines interest
UC102_7
The system shall display application to elderly after admin approval
UC102_8
The system shall allow elderly to approve or reject application
UC102_9
The system shall require rejection reason when elderly rejects application
UC102_10
The system shall forward approved application to youth for final confirmation
UC102_11
The system shall notify elderly when youth declines match
UC102_12

UC102_13
The system shall send confirmation notifications to both parties when match is confirmed


USE CASE NAME
UC-01: DECIDE YOUTH APPLICATION
USE CASE DESCRIPTION
This use case allows elderly to receive and respond to youth interest, participate in pre-match communication, and decide on formal application to establish companionship
PRE-CONDITION
Elderly has registered account
POST-CONDITION
Relationship is confirmed and enters Stage 1, Both parties receive confirmation


BASIC FLOW

BASIC FLOW
ACTOR (Elderly)
SYSTEM
1. Elderly begins process





2. System continues by activating UC_103: Complete Profile
3. System displays interest notification when youth expresses interest [M1: Msg Interest Received] [UC102_1]
4. Elderly selects action [A1: Accept Interest] [A2: Decline Interest]





5. System checks elderly's active pre-match count [C1: Elderly Pre-Match Limits] [A3: Elderly Hit Active Pre-Match Limit] [UC102_3]
6. System checks youth's active pre-match count [C2: Youth Pre-Match Limits] [A4: Youth Hit Active Pre-Match Limit] [UC102_4]
7. System continues by activating UC_104: Pre-Match Communication [UC102_5]
8. System displays application after admin approves [M2: Msg Application Received] [C3: Application Information] [UC102_7] [A5: Admin Rejects Application]
9. Elderly reviews application
10. Elderly selects action [A6: Approve Application] [A7: Reject Application]




11. System forwards approved application to youth for confirmation [UC102_10]
12. System displays match confirmation after youth confirms [M3: Msg Match Ready] [A8: Youth Declines Match]
13. Elderly confirms match




14. System updates relationship status to Stage 1 [UC102_12]
15. System sends confirmation to both parties [M4: Msg Welcome Stage 1] [UC102_13]
16. Use case ends
ALTERNATIVE FLOW
A1: Accept Interest
A1.1 Elderly selects "Accept Interest" [UC102_2]
A1.2 Continue to step BF:5
A2: Decline Interest
A2.1 Elderly selects "Decline Interest" [UC102_2]
A2.2 System notifies youth [M5: Msg Interest Declined By Elderly] [UC102_6]
A2.3 Use case ends
A3: Elderly Hit Active Pre-Match Limit
A3.1 System detects elderly has 5 active pre-matches [UC102_3]
A3.2 System displays error [M6: Err Elderly Limit] [UC102_3]
A3.3 Use case ends
A4: Youth Hit Active Pre-Match Limit
A4.1 System detects youth has 3 active pre-matches [UC102_4]
A4.2 System displays error [M7: Err Youth Limit] [UC102_4]
A4.3 Use case end
A5: Admin Rejects Application
A5.1 Admin reviews and rejects application
A5.2 System notifies youth with rejection reason
A5.3 Elderly does not receive application
A5.4 Use case ends
A6: Approve Application
A6.1 Elderly selects "Approve Application" [UC102_8]
A6.2 Continue to step BF:11
A7: Reject Application
A7.1 Elderly selects "Reject Application" [UC102_8]
A7.2 System displays rejection reason form [C4: Rejection Reason Requirements] [UC102_9]
A7.3 Elderly enters rejection reason [UC102_9]
A7.4 Elderly submits rejection [UC102_9]
A7.5 System notifies youth with rejection reason [M8: Msg Application Rejected By Elderly]
A7.6 Use case ends
A8: Youth Declines Match
A8.1 Youth declines final match confirmation
A8.2 System notifies elderly [M9: Msg Match Declined By Youth] [UC102_11]
A8.3 Use case ends
A9: Pre-Match Ends Unsuccessfully
A9.1 Either party ends pre-match communication before application
A9.2 System notifies both parties [M10: Msg Pre-Match Ended]
A9.3 Use case ends


MESSAGE
M1: Msg Interest Received = {[Youth Name], age [X], has expressed interest in connecting with you! View their profile to learn more and decide whether to accept}

M2: Msg Application Received = {[Youth Name] has completed the pre-match period and submitted a formal application to become your companion. Please review their application and motivation letter}

M3: Msg Match Ready = {[Youth Name] has confirmed the match! Please confirm to officially begin your companionship journey}

M4: Msg Welcome Stage 1 = {Congratulations! Your companionship with [Partner Name] is now official. Welcome to Stage 1: Getting to Know Each Other}

M5: Msg Interest Declined By Elderly = {[Elderly Name] has declined your interest. You can browse other profiles and express new interest}

M6: Err Elderly Limit = {You have reached the limit of 5 active pre-match conversations. Please end one conversation before accepting new interest}

M7: Err Youth Limit = {This youth is currently chatting with 3 elders and has reached their limit. Please try again later}

M8: Msg Application Rejected By Elderly = {[Elderly Name] has reviewed your application and decided not to proceed. Reason: [Rejection Reason]. You can apply to other elders}

M9: Msg Match Declined By Youth = {[Youth Name] has decided not to proceed with the match. You can continue reviewing other applications}

M10: Msg Pre-Match Ended = {Pre-match communication with [Name] has ended. You can accept other interest notifications}
CONSTRAINT
C1: Elderly Pre-Match Limits
5 active pre-match conversations simultaneously
C2: Youth Pre-Match Limits
Youth can have a maximum of 3 active pre-match conversations simultaneously.
Pending interests that have not yet been accepted do not count towards this limit.
C3: Application Information
Youth profile photo and personal information
Motivation letter (100-1000 characters)
Youth's availability for interaction
Youth's preferred communication methods
Youth's commitment level
Youth's personal expectations
What youth can offer
Pre-match conversation history summary
Application submission date and time
C4: Rejection Reason Requirements
Minimum 50 characters
Maximum 500 characters
Reason must be respectful and constructive
System validates length before submission
