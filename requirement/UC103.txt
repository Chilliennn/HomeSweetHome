USE CASE NAME
UC-103: COMPLETE PROFILE
USE CASE DESCRIPTION
This use case allows users (youth or elderly) to complete their profile information including age verification, real identity for verification, display identity for browsing, and preferences to enable matching features
PRE-CONDITION
User has registered account
User type is determined (Youth or Elderly) 
Profile is incomplete
POST-CONDITION
Profile is marked as complete, Matching features are enabled


BASIC FLOW

BASIC FLOW
ACTOR
SYSTEM
1. User begins profile completion





2. System displays profile completion form [M1: Msg Profile Form] [UC103_1]

3. System displays age verification requirement [M2: Msg Age Verification Required] [C1: Age Verification Requirements]
4. User initiates age verification process [UC103_2] [A16: User Cancels Verification]
5. User provides IC/identity document for verification [UC103_2] 





6. System validates age based on verification result [C2: Age Requirements] [A1: Invalid Age Youth] [A2: Invalid Age Elderly] [A15: MyDigitalID Verification Failed] [UC103_3] [UC103_4]

7. System marks age as verified

8. System displays real identity section [M3: Msg Real Identity Section] [C3: Real Identity Information]
9. User enters real personal information

10. User selects "Upload Real Photo" 




11. System displays photo upload interface [C4: Real Photo Requirements]
12. User uploads real photo [UC103_5]




13. System validates photo format and size [C4: Real Photo Requirements] [A3: Invalid Photo Format] [A4: Invalid Photo Size] [UC103_6]

14. System displays display identity section [M4: Msg Display Identity Section] [C5: Display Identity Information]
15. User enters display name [UC103_7]




16. System validates display name [C6: Display Name Requirements] [A5: Invalid Display Name] [UC103_9]

17. System displays avatar selection options [C7: Avatar Options] [UC103_10] [UC103_11]
18. User selects action [A6: Choose Default Avatar] [A7: Upload Custom Avatar]




19. System displays public profile information fields [C8: Public Profile Information]
20. User selects interests and hobbies [UC103_12]




21. System displays self-introduction field [C9: Self-Introduction Requirements]
22. User enters self-introduction [UC103_13]




23. System validates self-introduction length [C9: Self-Introduction Requirements] [A8: Invalid Introduction Length] [UC103_13]
24. System displays language preferences [C10: Language Options]

25. User selects language preferences [UC103_14]

26. User click “Submit Profile”




27. System validates all required fields are completed [A9: Incomplete Required Fields] [UC103_15]

28. System verifies age verification is completed [UC103_18] [A10: Age Not Verified]

33. System displays success message [M11: Msg Profile Complete]

34. Use case continue

CONSTRAINT
C1: Age Verification Requirements
Users cannot complete profile without successful age verification
Matching and adoption features are disabled until age is verified
User must take clear photo of Malaysian IC (MyKad) for verification
Age verification must be completed before profile submission
C2: Age Requirements
Youth users: Age must be between 18-40 years old (inclusive)
Elderly users: Age must be 60 years or above
C3: Real Identity Information 
This information is only visible to system admin and revealed to match partner after official relationship is confirmed, including :
Real full name 
Date of birth 
Gender 
IC/Passport number 
Real photo showing face clearly 
Phone number 
Email address 
Location 
C4: Real Photo Requirements
Maximum file size: 5MB
Photo must show face clearly without masks, sunglasses, or heavy filters
One real photo required for verification
Photo should be recent (taken within past 6 months)
Only visible to admin and matched partner after confirmation
C5: Display Identity Information
Display name
Avatar/Profile picture 
This information is visible during profile browsing and pre-match communication
Real identity is revealed only after official match confirmation
C6: Display Name Requirements
Minimum: 2 characters
Maximum: 20 characters
Can contain: Letters, numbers, spaces
Cannot contain: Special symbols 
Cannot be offensive or inappropriate
Must be unique within the system
System validates uniqueness before acceptance
C7: Avatar Options
Option 1: Default Avatars
pre-designed avatars available
Categories: Cartoon characters, Nature themes, Hobby-related, Cultural symbols
All avatars are friendly and appropriate
Avatars are culturally diverse to represent Malaysia's multicultural society
Option 2: Custom Upload
Maximum file size: 2MB 
Can be illustration, cartoon, or personal photo
Cannot be identical to real photo (to maintain pre-match anonymity)
Cannot contain offensive, violent, or inappropriate content
C8: Public Profile Information (Always Visible)
Interests and hobbies (minimum 3, maximum 10 selections)
Self-introduction (50-500 characters, required)
Brief background (optional, 100-300 characters)
Preferred communication style (required)
Language preferences (required, at least one)
Availability for interaction (required)
What looking for in a companion (optional, 100-300 characters)
This information is visible to all users during profile browsing
C9: Self-Introduction Requirements
Minimum: 50 characters
Maximum: 500 characters
Should describe personality, life experience, or what you're looking for in a companion
System does not validate content quality
Visible to all users
Can be edited after profile completion
C10: Language Options
Bahasa Malaysia
English
Mandarin
Tamil
Other 
At least one language must be selected
Helps match users with common languages for better communication
